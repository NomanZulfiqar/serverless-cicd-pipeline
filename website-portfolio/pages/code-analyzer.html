Complete Setup Instructions

Prerequisites

  • Git: Must be installed and configured on your system
  • Python 3.7+: Required for the hook script execution
  • Git Repository: Your project must be a Git repository (run git init if not)
  • Internet Connection: Required for code analysis features

Step 1: Download Required Files

Download both files using the buttons above:

  • pre-commit - The main hook script
  • pre_commit_hook.py - The Python analysis script

Step 2: Prepare Your Repository

Navigate to your Git repository's root directory:

cd /path/to/your/repository

Ensure you're in the correct directory by checking for the .git folder:

ls -la | grep .git

Step 3: Install the Hook Files

Move the downloaded files to your repository's .git/hooks/ directory:

For Windows:

copy pre-commit .git\hooks\
copy pre_commit_hook.py .git\hooks\

For Mac/Linux:

mv pre-commit .git/hooks/
mv pre_commit_hook.py .git/hooks/

Step 4: Set File Permissions

Make the hook files executable:

For Mac/Linux:

chmod +x .git/hooks/pre-commit
chmod +x .git/hooks/pre_commit_hook.py

Step 5: Test the Setup

Verify the installation by making a test commit:

git add .
git commit -m "Test: Verify pre-commit hook installation"

If successful, you should see analysis output in your terminal. If issues are found, the commit will be blocked until resolved.

How It Works

  • Automatic Analysis: Every commit triggers code quality analysis
  • Quality Checks: Identifies potential bugs, security issues, and code improvements
  • Blocking Commits: Prevents commits with critical issues from being pushed
  • Detailed Feedback: Provides specific recommendations for code improvements

Troubleshooting

  • Hook Not Running: Check file permissions and ensure files are in .git/hooks/
  • Python Errors: Verify Python 3.7+ is installed and accessible
  • Permission Denied: Run chmod +x commands on Mac/Linux
  • File Not Found: Ensure you're in the correct repository directory
